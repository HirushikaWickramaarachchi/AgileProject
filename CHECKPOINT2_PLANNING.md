# Checkpoint 2: Application Planning (ClubSync)

## 1) Proposed Application
ClubSync is a student club management platform for universities. It helps students discover and join clubs, track events, and manage participation. It also provides admin tools for event and member management.

## 2) User Stories (10+)
1. As a student, I want to sign up for an account so that I can access club features.
2. As a student, I want to log in securely so that I can view personalized club content.
3. As a student, I want to browse all clubs so that I can find communities that match my interests.
4. As a student, I want to view detailed club information so that I can decide whether to join.
5. As a student, I want to view upcoming events so that I can plan my participation.
6. As a student, I want to register for an event so that I can attend and receive updates.
7. As an admin, I want to create new events so that clubs can publish activities for members.
8. As an admin, I want to edit existing events so that event details stay accurate.
9. As an admin, I want to manage member roles so that trusted users can moderate club activities.
10. As an admin, I want to filter and search members so that I can quickly find and manage specific users.
11. As an admin, I want to review dashboard metrics so that I can monitor engagement and operations.
12. As a club member, I want to see event status and updates so that I can stay informed about changes.

## 3) Main Pages
- Home page
- Signup page
- Login page
- Club list page
- Club details page (My Club)
- Event details page
- Admin dashboard page
- Admin create/edit event page
- Admin manage members page

## 4) CSS Framework Decision
The project uses **Bootstrap 5** for responsive layout, UI components, and consistent styling.  
Custom CSS is used to match the ClubSync visual identity and provide branded page designs.

## 5) Team Organization (for checkpoint discussion)
- Weekly planning sync: feature planning, task assignment, and blockers.
- Mid-week progress check: update implementation status.
- GitHub workflow: branch-based work with pull requests and review before merge.
- Task split: authentication, clubs/events UI, admin functionality, and documentation/testing.

## 6) Scope Note for Checkpoint 2
At this stage, the project focuses on UI planning and static pages.  
Backend implementation (Flask routes, database integration) will be completed after relevant class coverage.
